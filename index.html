Cache Poisoning Redirect - Validation Page

This page is intentionally hosted by the security researcher for the purpose of demonstrating the impact of a Shared CDN Cache Poisoning vulnerability. If you are seeing this page, it means your request was redirected here as a result of a previously poisoned cache entry.

Final Destination (After Redirect)

Purpose of This Page

This page serves as evidence that a redirect response, influenced by attacker-controlled input, was cached by the CDN and delivered to normal users without requiring the malicious header to be present. This confirms:

  • The redirect was stored as a shared cache variant.
  • Visitors received a poisoned redirect response.
  • The CDN did not include relevant request headers in its cache key.

Technical Notes

For validation purposes, the researcher can provide:

  • The poisoning request sent to the origin.
  • The cached response showing x-cache: HIT.
  • The redirect chain leading to this page.
  • Timestamps, headers, and request logs if required.

No malicious content is delivered from this domain. This page is used exclusively for vulnerability validation under an authorized security testing program.
